Usage

StandaloneHost can be used in two ways. The first way is to permanently host a V2 Audio Unit soft synth by right clicking on StandaloneHost.app (in the /Applications folder) and selecting Duplicate. Then rename the duplicate copy to match the file name of the soft synth you wish to host. The file names for the soft synths installed on your Mac can found in the /Library/Audio/Plug-Ins/Components folder.

For example, to host the freeware OB-Xd soft synth, rename a copy of StandaloneHost.app to OB-Xd.app.

Make sure the new name exactly matches the file name of the soft synth bundle in the Components folder (without the .component extension), including any spaces and capitalizations. If there are any typos in the new name, StandaloneHost will load the Apple DLS Music Device instead. The Apple DLS Music Device will also be loaded if StandaloneHost is incompatible with the desired synth.

The second way is to start StandaloneHost without renaming it, then selecting a synth using the Operations...Select Synth menu. This will allow you to try synths without duplicating and renaming StandaloneHost.

Check the compatibility table for incompatible synths and any other known issues.
